What is Quantum Computing?

⚛️

Qubits

Classical computers use bits that are either 0 or 1. Quantum computers use qubits, which can be both 0 and 1 at the same time — a property called superposition.

🌊

Interference

Like waves in water, quantum amplitudes can add together or cancel out. Quantum algorithms are designed so that wrong answers cancel and right answers amplify.

📏

Measurement

When you measure a qubit, superposition collapses to a definite 0 or 1. The trick is to arrange interference first so the right answer is most probable.

⚡

Quantum Advantage

For certain problems, quantum computers can find answers in exponentially fewer steps than the best classical algorithms — that's the advantage we demonstrate here.

The Algorithms

Two quantum algorithms that dramatically outperform their classical counterparts — both implemented from scratch in Java.

Algorithm 01

Deutsch-Jozsa

The Problem

You're given a mystery function f that takes a binary input and outputs 0 or 1. The function is guaranteed to be one of two types:

  • Constant — outputs the same value (all 0s or all 1s) no matter what you put in
  • Balanced — outputs 0 for exactly half of inputs and 1 for the other half

Your job: figure out which type it is.

Classical
513
queries needed in the worst case for a 10-bit input (2n-1 + 1)
VS
Quantum
1
query. Always. No matter how big the input is.

How it works — step by step

1
Initialize — Start all qubits in state |0⟩
2
Superposition — Apply Hadamard gates to create an equal mix of all possible inputs at once
3
Oracle query — Run the mystery function once. It flips the phase (sign) of states where f(x) = 1
4
Interference — Apply Hadamard again. Constant functions interfere constructively at |00⟩, balanced functions don't
5
Measure — If you measure |00...0⟩, the function is constant. Anything else means balanced.

Actual simulator output

Step 1: Initialize to |0⟩²
  |00⟩: 1.0000  (probability: 1.0000)

Step 2: Apply Hadamard — superposition of all inputs
  |00⟩: 0.5000  |01⟩: 0.5000  |10⟩: 0.5000  |11⟩: 0.5000

Step 3: Apply oracle (ONE QUERY)
  [balanced fn] |00⟩: +0.5000  |01⟩: +0.5000  |10⟩: -0.5000  |11⟩: -0.5000
  ↑ phase flips on states where f(x) = 1

Step 4: Apply Hadamard again — interference
  |10⟩: 1.0000  (probability: 1.0000)
  ↑ did NOT collapse to |00⟩ → BALANCED ✓

Statistical test: 1000/1000 runs correct (100% accuracy)
Algorithm 02

Bernstein-Vazirani

The Problem

There's a hidden secret bit string s. You have access to an oracle that computes:

f(x) = s · x (mod 2)

This is the bitwise dot product — for each bit position, multiply the corresponding bits of s and x, then XOR everything together. Your job: find the entire secret string s.

Classical
n
queries needed — one carefully chosen query per bit of the secret
VS
Quantum
1
query recovers the entire secret string, no matter how long it is

How it works — step by step

1
Initialize — Start all qubits in state |0⟩
2
Superposition — Hadamard gates create a uniform superposition of all possible inputs
3
Oracle query — The oracle computes the dot product in superposition, encoding the secret as phase flips
4
Interference — A second round of Hadamard gates transforms the phase information back into amplitude
5
Measure — The measurement result is the secret string. Directly.

Actual simulator output — recovering secret "101"

Step 2: Superposition of all 3-qubit inputs
  |000⟩: 0.3536  |001⟩: 0.3536  |010⟩: 0.3536  |011⟩: 0.3536
  |100⟩: 0.3536  |101⟩: 0.3536  |110⟩: 0.3536  |111⟩: 0.3536

Step 3: Oracle applies phase flips based on s · x
  |000⟩: +0.3536  |001⟩: -0.3536  |010⟩: +0.3536  |011⟩: -0.3536
  |100⟩: -0.3536  |101⟩: +0.3536  |110⟩: -0.3536  |111⟩: +0.3536

Step 4: Hadamard — interference collapses everything
  |101⟩: 1.0000  (probability: 1.0000)
  ↑ measurement gives us exactly the secret ✓

Recovery rate across 1000 runs: 100.0%

Built from scratch in Java

The Simulator

Every component implemented manually — no quantum libraries, no shortcuts. Here's how the code works.

How to Run It

1

Set up the file structure

Place all .java files inside a folder called src/quantum/

2

Compile

javac -d bin src/quantum/*.java
3

Run

java -cp bin quantum.Main

Project Structure

Complex.java

Represents quantum amplitudes as complex numbers. Stores real and imaginary parts, handles multiplication, and computes |amplitude|² for probability.

Complex amp = new Complex(0.5, 0);
double prob = amp.magnitudeSquared();
// prob = 0.25 (25% chance)
QuantumState.java

Stores an n-qubit state as an array of 2ⁿ complex amplitudes. For 2 qubits, there are 4 possible states: |00⟩, |01⟩, |10⟩, |11⟩.

QuantumState state = new QuantumState(2);
state.setToZero(); // starts at |00⟩
state.print();     // shows all amplitudes
QuantumGate.java

Implements the Hadamard gate (creates superposition) and the oracle (encodes the hidden function as phase flips on the quantum state).

// Create superposition
QuantumGate.applyHadamard(state, 0);

// Encode function as phase flips
QuantumGate.applyPhaseFlip(state, oracle);
QuantumMeasurement.java

Simulates measurement by building a cumulative probability distribution from the amplitudes and drawing a random outcome — just like real quantum measurement.

int result = QuantumMeasurement.measure(state);
// Each state |x⟩ has P = |amplitude[x]|²
// Returns the collapsed outcome
DeutschJozsa.java

Full implementation of the Deutsch-Jozsa algorithm. Takes the number of qubits and an oracle function, runs the algorithm, and returns whether the function is constant or balanced.

QuantumGate.OracleFunction f = x -> 0; // constant
boolean balanced = DeutschJozsa.run(2, f);
// balanced = false ✓
BernsteinVazirani.java

Full implementation of Bernstein-Vazirani. Given a dot-product oracle encoding a secret string, recovers the entire secret in a single quantum query.

// Oracle encodes secret = 5 (binary: 101)
int recovered = BernsteinVazirani.run(3, oracle);
// recovered = 5 → binary "101" ✓

Deutsch-Jozsa 10 bits 513 1 513×
Bernstein-Vazirani 2 bits 2 1 2×

Quantum always uses exactly 1 oracle query. Classical worst case for Deutsch-Jozsa is 2n-1+1; for Bernstein-Vazirani it's n (one query per bit).
